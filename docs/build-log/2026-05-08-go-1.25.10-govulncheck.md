# Go 1.25.10 Govulncheck Follow-Up

**Status:** in review
**Date:** 2026-05-08

## What

Bump the pinned Go toolchain from 1.25.9 to 1.25.10 in the module and CI/release workflows.

## Why

`govulncheck` started flagging `GO-2026-4971` against the Go 1.25.9 standard library (`net` on Windows via `Dial`/`LookupPort`). The CI dependency-verification job now fails until the repo pins the fixed patch release.

## Changes

- Updated `go.mod` from `go 1.25.9` to `go 1.25.10`.
- Updated `.github/workflows/ci.yml` to use `go-version: "1.25.10"` for dependency verification, lint, and test jobs.
- Updated `.github/workflows/release.yml` to use `go-version: "1.25.10"` for release verification and packaging jobs.

## Impact

- Keeps `govulncheck` green against the current Go standard library vulnerability feed.
- Preserves the existing 1.25.x baseline with the smallest possible toolchain change.

## Quality

- `make check`
- `GOCACHE=/tmp/go-build GOLANGCI_LINT_CACHE=/tmp/golangci-lint make lint`
- `GOCACHE=/tmp/go-build make test`
- `GOCACHE=/tmp/go-build make build`
